metabase 101

Metabase is where we build every report and dashboard for Knight Aerospace departments. it connects directly to the Cetec MySQL database — so anything you can query in SQL, you can visualize here.

what is metabase

📝 SQL editor

Write raw MySQL queries and see results instantly — this is how we build most of our reports

📊 visualizations

Turn query results into bar charts, line graphs, tables, numbers, and more

📋 dashboards

Combine multiple charts into one page — each department has their own dashboard

🔍 filters

Add date pickers, dropdowns, and text inputs so users can filter reports without touching SQL

writing a SQL question

In Metabase, a "question" is just a saved SQL query with a visualization attached. here's the basic workflow:

StepWhat to Do
01Click New → SQL query and make sure the database is set to Cetec
02Write your query in the editor — use CTEs to keep it readable
03Hit Run (or Ctrl+Enter) to see results
04Click the Visualization button at the bottom to pick a chart type
05Click Save, give it a clear name, and save to your personal collection first
06When it's ready to share, move it to the right team collection
adding filters with variables

Metabase has a special syntax for adding interactive filters to SQL queries. this lets users filter reports without editing SQL.

optional filter syntax: wrap a filter in [[ ]] double brackets to make it optional — if the user doesn't set it, the filter is ignored and all rows return.
SyntaxWhat It DoesExample
{{variable}} Required filter — always applied WHERE vendor_id = {{vendor}}
[[AND col = {{var}}]] Optional filter — ignored if not set [[AND partno = {{part}}]]
{{date}} with date type Date picker filter WHERE po_date >= {{start_date}}
visualization types & when to use them
Chart TypeBest For
TableDetailed row-level data, reference reports, anything with many columns
Bar chartComparing values across categories (spend by vendor, defects by part)
Line chartTrends over time (monthly spend, weekly scrap rates)
Single numberKPIs — one big stat on a dashboard (total open POs, on-time %)
Row chartRankings — like a bar chart but horizontal, great for long category names
Pivot tableCross-tab summaries — rows and columns both grouping data
tips from Aliya

save early, save often

Metabase doesn't autosave. if you close the tab mid-query, it's gone. save drafts to your personal collection constantly.

use CTEs, not subqueries

Metabase renders CTEs cleanly and they're way easier to debug. if your query has more than 2 joins, it probably needs a CTE.

check your JOINs

A wrong JOIN in Metabase doesn't error — it just returns wrong numbers silently. always sanity check row counts after adding a join.

naming convention

Name questions like: [Team] — Report Name — Date. Example: Procurement — Vendor OTD — 2026. Makes searching way easier.

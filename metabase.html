metabase Guide

Metabase is where we build every report and dashboard for Knight Aerospace departments. it connects directly to the Cetec MySQL database. So anything you can query in SQL, you can visualize here.

what is metabase

SQL editor

Write GUI or raw MySQL queries and visualize them all in the application. This is how we build most of our reports

Visualizations

Turn query results into bar charts, line graphs, tables, numbers, and more

Dashboards

Combine multiple charts/reports into one page. (We should probably organize reports based on department)

Filters

Add date pickers, dropdowns, and text inputs so users can filter reports without touching the code

writing a SQL question

In Metabase, a "question" is just a saved SQL query with a visualization attached. Basically the no-code query builder. Here's the basic workflow:

StepWhat to Do
01Click New → SQL query and make sure the database is set to Cetec
02Write your query in the editor
03Hit Run (or Ctrl+Enter) to see results
04Click the Visualization button at the bottom to pick a chart type
05Click Save, give it a clear name, and save to your personal collection first
06When it's ready to share, move it to the right team collection
adding filters with variables

Metabase has a special syntax for adding interactive filters to SQL queries. this lets users filter reports without editing SQL.

optional filter syntax: wrap a filter in [[ ]] double brackets to make it optional so that if the user doesn't set it, the filter is ignored and all rows return.
SyntaxWhat It DoesExample
{{variable}} Required filter, always applied WHERE vendor_id = {{vendor}}
[[AND col = {{var}}]] Optional filter, ignored if not set [[AND partno = {{part}}]]
{{date}} with date type Date picker filter WHERE po_date >= {{start_date}}
Tips

save early, save often

Metabase doesn't autosave. So, if you close the tab mid-query, it's gone. Save drafts to your personal collection constantly.

use CTEs, not subqueries

Metabase renders CTEs cleanly and they're way easier to debug. They are a temporary, named result set defined within a query using the WITH keyword. If your query has more than 2 joins, it probably needs a CTE.
For example:
WITH cte_name AS (
 SELECT
 FROM
 WHERE
)
SELECT *
FROM cte_name;.

check your JOINs

A wrong JOIN doesn't error, it just returns wrong numbers. So, always sanity check the row counts after adding a join.
